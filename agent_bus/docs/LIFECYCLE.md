# Lifecycle

1. Draft a task from `templates/TASK_TEMPLATE.md`.
2. Set `status: ready` when approved.
3. The assignee claims the task and updates it as work progresses.
4. The assignee writes a matching result file from `templates/RESULT_TEMPLATE.md`.
5. The human supervisor decides whether to archive the pair.

The lifecycle stays file-driven on purpose so the repo can be mirrored, reviewed, and audited without hidden state.
